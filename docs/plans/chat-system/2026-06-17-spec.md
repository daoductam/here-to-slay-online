## 1. Goal & Requirements
- **Goal:** Add real-time text chatting capability inside lobbies and active games to enhance player interaction.
- **Requirements:**
  - Chat must be accessible on both the Lobby page (`/lobby`) and the Game board page (`/game`).
  - Chat UI should be a collapsible sidebar (sliding in/out from the right side of the screen) toggled via a button with an unread messages badge.
  - Chat history should persist when players navigate between Lobby and Game screens or reconnect due to connection drop.
  - Play a short sound effect (`sfx_chat.mp3`) when a new message is received, adhering to the user's SFX volume and enable/disable preferences.

---

## 2. Technical Approach & Architecture

### Socket Events Contract
We will define the following socket communication interfaces:

- **Client to Server:**
  - `chat:send` — Sent when a user submits a message:
    ```typescript
    (roomId: string, userId: string, text: string) => void
    ```

- **Server to Client:**
  - `chat:message` — Broadcasts a new message to everyone in the room:
    ```typescript
    (message: ChatMessage) => void
    ```
  - `chat:history` — Sends all existing messages in the room upon client joining/reconnection:
    ```typescript
    (history: ChatMessage[]) => void
    ```

### Data Structures
We define the message format in `types.ts` (both client and server):
```typescript
export interface ChatMessage {
  id: string;        // Unique message ID (uuid/nanoid)
  sender: string;    // Username of sender
  text: string;      // Message text
  timestamp: number; // Date.now() timestamp
  isSystem?: boolean; // True if message is a system notification (e.g. joins, leaves)
}
```

On the Server:
- Each room object in `rooms` will store a `chatHistory` array of type `ChatMessage[]`.
- Max history capacity is capped at 100 messages to prevent excessive memory usage.

## 3. UI/UX Design (Premium Fantasy Theme)
- A sliding drawer panel on the right side of the screen.
- Layout:
  - Header: Title ("Trò chuyện") + Close Button.
  - Body: Scrollable list of message bubbles.
    - System alerts styled with italicized orange-brown text.
    - My messages styled with a distinct border/background color (e.g., orange-tinted dark border) and aligned to the right.
    - Other players' messages aligned to the left with their usernames shown above bubbles.
  - Footer: Chat input text box and a Send button (represented with a send icon).
- Chat Toggle Button:
  - An icon button with a chat bubble icon.
  - Positioned next to other navigation/settings buttons (e.g., inside MenuButtons or TopMenu).
  - Displays a red badge with the count of unread messages when the chat sidebar is collapsed.

---

## 4. Sound Cue Integration
- Add a new sound asset file `sfx_chat.mp3` to `client/public/audio/`.
- Wire `playSFX('chat')` inside the `chat:message` listener on the client.
- Condition: The SFX only triggers if the message is from another player AND either the chat sidebar is currently collapsed or the document is not focused.

---

## 5. Verification Plan
- **Real-Time Delivery:** Open two browser tabs, join the same lobby, type messages in tab A, verify they appear instantly in tab B.
- **Persistence Check:** Send messages in Lobby, start the game, verify that previous messages display correctly in the Game's chat sidebar.
- **Audio Cue Check:** Close the chat panel in tab B, send a message from tab A, verify the notification SFX plays.
