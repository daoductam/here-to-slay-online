# 1. Context

**Problem:**
Toàn bộ 107 ảnh PNG (~138 MB) được load đồng thời khi vào game, và `Game.tsx` chặn UI cho đến khi 100% ảnh tải xong (`loadedImages === everyCard.length`). Điều này khiến màn hình loading kéo dài từ 30–60 giây trên kết nối bình thường.

**Affected Modules:**
- `scripts/convert-to-webp.js` — script offline 1 lần để convert PNG → WebP
- `client/src/cards.ts` — nguồn dữ liệu toàn bộ card, cần tách thành critical / deferred
- `client/src/helpers/getImage.ts` — helper sinh đường dẫn ảnh, cần trỏ sang `.webp`
- `client/src/pages/Game.tsx` — loading logic, cần unblock sớm hơn
- `client/src/App.tsx` — preload tại home page, cần cập nhật theo nhóm mới

**Non-Goals:**
- Không dùng CDN hoặc service worker
- Không áp dụng IntersectionObserver (on-demand per-card)
- Không xóa file PNG gốc
- Không thay đổi bất kỳ logic game nào ngoài phần loading

# 2. Constraints

**Language:** `TypeScript (React), JavaScript (script)`
**Architecture:** `Client-Server, React SPA`

**Rules:**
- Không sửa file ngoài danh sách Artifact Registry
- `sharp` chỉ là devDependency — không ship lên production client
- File WebP phải nằm cùng thư mục với file PNG gốc (giữ nguyên cấu trúc)
- Extension `.webp` thay thế `.png` nhất quán trong toàn client code

**Performance Budgets:**
- Thời gian từ mở game đến hiển thị UI: < 8 giây (kết nối bình thường)
- Phase 1 critical assets: < 5 MB WebP tổng cộng

# 3. Conventions

**Naming:**
- Constants: `camelCase` — `criticalAssets`, `deferredAssets`
- State: `loadedCritical` thay thế `loadedImages`

**Quy tắc ảnh:**
- Tất cả đường dẫn ảnh trong client dùng `.webp` sau khi hoàn thành
- `getImage()` là điểm duy nhất sinh đường dẫn — không hardcode `.png` ở nơi khác

**Error Handling:**
- Script convert: log lỗi từng file, tiếp tục xử lý file khác
- Runtime: không throw — nếu ảnh lỗi browser tự hiển thị broken image

# 4. Contracts

## Data Structures

```
AssetGroup {
  criticalAssets: (AnyCard | string)[]
    — ~15 items: card back, icons, modifier, challenge, SVG
    — load trước, unblock game UI khi hoàn tất

  deferredAssets: (AnyCard | string)[]
    — ~92 items: hero, monster, item, magic, large, help
    — load sau trong background
}
```

## Interface: `getImage(card: AnyCard): string`

```
getImage(card: AnyCard): string
  - pre: card không null, card.type hợp lệ
  - post: trả về URL dạng /assets/<type>/.../<name>.webp
  - không throw — trả về undefined nếu card không hợp lệ (giữ nguyên cũ)
```

## Interface: `convert-to-webp script`

```
run(): void
  - pre: client/public/assets/ tồn tại, sharp đã cài
  - post: mỗi file .png có 1 file .webp cùng tên cùng thư mục
  - on error per file: log warning, tiếp tục
  - không xóa file .png gốc
```

# 5. Target Architecture

**Components:**
- `convert-to-webp.js` — script Node.js chạy offline, convert tất cả PNG → WebP
- `cards.ts` — export thêm `criticalAssets` và `deferredAssets` bên cạnh `everyCard`
- `getImage.ts` — sinh URL `.webp` thay vì `.png`
- `Game.tsx` — unblock UI sau khi `criticalAssets` load xong, load `deferredAssets` ngầm
- `App.tsx` — cập nhật preload loop theo nhóm mới

**Interaction Flow:**

```
[Offline — 1 lần]
node scripts/convert-to-webp.js
  → sharp(png).webp({quality:85}).toFile(webp) cho từng PNG
  → log ✓/✗ per file

[Runtime — Game.tsx]
useEffect mount:
  → Loop criticalAssets (~15 ảnh)
       → setLoadedCritical(v => v+1) mỗi onload
  → Loop deferredAssets (~92 ảnh)
       → fire-and-forget (không track)

render:
  loadedCritical < criticalAssets.length → loading screen
  loadedCritical === criticalAssets.length → game UI ✓ (deferred còn loading ngầm)
```

**Key Decisions:**
- Ngưỡng unblock thu từ `everyCard.length` (107) xuống `criticalAssets.length` (~15)
- `deferredAssets` vẫn preload ngay (không lazy) để sẵn sàng khi gameplay cần
- PNG gốc giữ lại để rollback bằng 1 dòng code

# 8. Task Specifications

## TASK-001: Viết script convert PNG → WebP

**Description:**
Tạo `scripts/convert-to-webp.js` dùng `sharp` để duyệt đệ quy `client/public/assets/`, tìm tất cả `.png`, tạo `.webp` cùng thư mục. Thêm `sharp` vào devDependencies.

**Input:** None (root task)

**Output:** `scripts/convert-to-webp.js` — consumed by TASK-004

**Files:**
- `scripts/convert-to-webp.js` — **create**
- `client/package.json` — **modify**: thêm `sharp` devDep + script `convert:webp`

**Acceptance Criteria:**
- [ ] Mỗi `.png` trong `/assets/` có file `.webp` tương ứng sau khi chạy
- [ ] `.webp` nhỏ hơn `.png` gốc ít nhất 50%
- [ ] Script không crash khi gặp file lỗi
- [ ] Tổng dung lượng WebP < 50 MB
- [ ] Idempotent — chạy lại không lỗi, overwrite WebP cũ

## TASK-003: Cập nhật loading logic `Game.tsx` + `App.tsx`

**Description:**
Trong `Game.tsx`, thay `loadedImages/everyCard.length` bằng `loadedCritical/criticalAssets.length`. Load `deferredAssets` fire-and-forget ngay sau. Cập nhật `App.tsx` tương tự.

**Input:** Từ TASK-002: `criticalAssets`, `deferredAssets`

**Output:** `Game.tsx` + `App.tsx` — consumed by TASK-004

**Files:**
- `client/src/pages/Game.tsx` — **modify**
- `client/src/App.tsx` — **modify**

**Key changes trong Game.tsx:**
```typescript
// Trước:
const [loadedImages, setLoadedImages] = useState(0);
// ...
for (let card of everyCard) { img.onload = () => setLoadedImages(v => v+1); }
// render: loadedImages === everyCard.length ? <Game/> : <Loading/>

// Sau:
const [loadedCritical, setLoadedCritical] = useState(0);
// ...
for (let card of criticalAssets) { img.onload = () => setLoadedCritical(v => v+1); }
for (let card of deferredAssets) { /* fire-and-forget */ }
// render: loadedCritical === criticalAssets.length ? <Game/> : <Loading/>
```

**Acceptance Criteria:**
- [ ] Game UI hiển thị sau khi chỉ critical assets load xong
- [ ] Progress bar đúng: `(loadedCritical / criticalAssets.length) * 25vw`
- [ ] `deferredAssets` vẫn fetch trong background (Network tab)
- [ ] Không có TypeScript error mới
- [ ] Ảnh hero/monster vẫn hiển thị đúng trong gameplay

# 9. Edge Cases

| # | Scenario | Expected Behavior | Handled In |
|---|---|---|---|
| 1 | File PNG bị corrupt khi convert | Log lỗi, bỏ qua, tiếp tục | TASK-001 |
| 2 | `circle-star.svg` không phải PNG | Script bỏ qua, SVG dùng như cũ | TASK-001, TASK-002 |
| 3 | Ảnh deferred chưa load khi card được đánh ra | Browser hiển thị khi fetch xong (đã bắt đầu ngầm) | TASK-003 |
| 4 | Script chạy lần 2 (đã có WebP) | Overwrite, idempotent, không lỗi | TASK-001 |
