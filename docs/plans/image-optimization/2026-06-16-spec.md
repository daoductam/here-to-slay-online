# Image Optimization: WebP Conversion + Two-Phase Loading

## Problem

Hiện tại game load **107 ảnh PNG** với tổng dung lượng ~**138 MB**. `Game.tsx` chặn toàn bộ UI cho đến khi 100% ảnh được tải xong (`loadedImages === everyCard.length`), gây ra màn hình loading kéo dài không cần thiết.

## Root Cause

1. **Định dạng ảnh không tối ưu**: Toàn bộ ảnh dùng PNG, trong khi WebP cho chất lượng tương đương với dung lượng nhỏ hơn 60–80%.
2. **Blocking load strategy**: Tất cả 107 ảnh phải load xong mới hiện game, kể cả các ảnh hero/monster chưa cần dùng ngay.

## Solution

### Phần A — Offline: Convert PNG → WebP (1 lần)

- Viết script Node.js dùng thư viện `sharp` để convert toàn bộ ảnh trong `client/public/assets/` từ PNG → WebP.
- File gốc PNG được giữ nguyên (không xóa), WebP được tạo song song cùng thư mục.
- `getImage()` helper được cập nhật để trả về đường dẫn `.webp` thay vì `.png`.
- **Ước tính**: 138 MB → ~25–40 MB (tiết kiệm ~70%).

### Phần B — Runtime: Two-Phase Loading

Tách `everyCard` thành hai nhóm:

**Phase 1 — Critical assets** (~20 items, ~3–5 MB WebP):
- Card back (`back-creme.webp`)
- 7 class icons (`bard`, `fighter`, `guardian`, `ranger`, `thief`, `wizard`, `hero`)
- `circle-star.svg`
- Modifier cards (5 ảnh)
- Challenge card (1 ảnh)

→ Khi phase 1 hoàn tất, game UI được hiển thị.

**Phase 2 — Deferred assets** (~87 items, ~20–35 MB WebP):
- Toàn bộ hero cards (48 ảnh)
- Large/Monster cards (21 ảnh)
- Item cards (12 ảnh)
- Magic cards (8 ảnh)
- Help cards (2 ảnh)

→ Load ngầm trong background sau khi game đã hiển thị. Nếu ảnh chưa load kịp khi render, card back được dùng làm placeholder tạm thời.

## Files Affected

| File | Thay đổi |
|---|---|
| `scripts/convert-to-webp.js` | [NEW] Script offline convert PNG → WebP |
| `client/src/helpers/getImage.ts` | Cập nhật extension từ `.png` → `.webp` |
| `client/src/cards.ts` | Tách `everyCard` thành `criticalAssets` + `deferredAssets` |
| `client/src/pages/Game.tsx` | Cập nhật loading logic: unblock sau phase 1 |
| `client/src/App.tsx` | Cập nhật preload theo nhóm mới |
| `server/package.json` hoặc `package.json` | Thêm `sharp` vào devDependencies |

## Success Criteria

- Thời gian từ mở game đến show UI giảm từ ~30–60s xuống còn ~3–8s (trên kết nối bình thường).
- Không có lỗi ảnh vỡ sau khi chuyển sang WebP.
- Toàn bộ ảnh vẫn hiển thị đúng trong gameplay (hero, monster, item, magic).

## Out of Scope

- Không dùng CDN hoặc service worker trong phiên bản này.
- Không áp dụng IntersectionObserver (on-demand loading).
- Không xóa file PNG gốc (giữ lại để có thể rollback).
