# Design Spec: Player Count Selection (2 to 5 players)

## Overview
Currently, the game only supports starting matches with a hardcoded minimum of 3 players. This feature adds the ability for a room creator to choose the player count (between 2 and 5 players) during room creation. The lobby will only start once it is fully populated with the selected number of players and all players are ready. Joining is also restricted once the lobby is full.

## User Decisions
- **Lobby Start Flow**: The game starts only when the lobby is fully populated (joined players count matches target count) and all players are ready.

## Proposed Changes

### Frontend (`client/`)
- **`client/src/types.ts`**: Add `targetPlayers` to `GameState['match']`.
- **`client/src/App.tsx`**:
  - Add a `<select>` dropdown for player count in the room creation form (options: 2, 3, 4, 5. Default: 3).
  - Update `createRoom()` to send `targetPlayers` in the POST request body.
  - Update room listing to display players count as `Players: [joined]/[target]` (e.g. `Players: 2/4`).
- **`client/src/pages/Lobby.tsx`**:
  - Render blank player loading slots based on `matchState.targetPlayers - matchState.players.length` instead of the hardcoded `3 - matchState.players.length`.

### Backend (`server/`)
- **`server/src/types.ts`**: Add `targetPlayers` to `GameState['match']` and `Room` types.
- **`server/src/controllers/express/homeController.ts`**:
  - Initialize the room with the selected `targetPlayers` from the request body (defaulting to 3 if not provided).
  - Store this `targetPlayers` in `gameState.match.targetPlayers`.
  - In `joinRoom()`, retrieve the target player count and reject any join attempts if the room is already full (`room.numPlayers >= targetPlayers`).
  - Update `getRooms()` API to return room status containing both the joined count and target count.
- **`server/src/controllers/socketio/lobby/lobby.ts`**:
  - Replace the hardcoded `rooms[roomId].numPlayers >= 3` checks in the ready/start-match flows with a check for `rooms[roomId].numPlayers === rooms[roomId].state.match.targetPlayers`.
